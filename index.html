WTF: HTML II - Atrybuty i nowe znaczniki w HTML

Przypomnijmy - budowa znacznika:

<znacznik atrybut="wartość">treść</znacznik>

<br>* - pozwala wstawić nową linię
/złamać tekst do nowej linii

*<br/> - starsza wersja zapisu, także prawidłowa

ELEMENTY DO WYRÓŻNIANIA TEKSTU
<b> vs. <strong> vs. <i> vs. <em>

<b> - pogrubiony tekst (BOLD w Wordzie)

<i> - kursywa (Italic w Wordzie)

Wyróżnienie znaczeniowe

<strong> - pogrubiony tekst (podkreślenie istotności i powagi frazy w nim zawartej)

<em>> - pochylony tekst (podkreśla, że tekst może mieć inne znaczenie, np. może wyrażać ironię)

Wyróżnienie wizualne

<b> i <i> - nie mają dużego znaczenia semantycznego - służą do wyróżnienia fragmentu z tekstu otaczającego

I extra, do uzupełnienia Word-owego trio:

<u> - podkreślenie (Underline w Wordzie)

ŁĄCZA (LINKI) WEWNĄTRZ I NA ZEWNĄTRZ DOKUMENTU
znacznik <a>

<a href= "" target=""> opis łącza </a>

atrybut href - adres strony www/lokalizacja w dokumencie (id)
atrybut target - cel

wartości atrybutu target: _self (to samo okno) | _parent | _top | _blank (nowe okno/zakładka)| <XML-Name>
domyślna wartość: _self

Adres www - wpisujemy adres z http:// lub https://

Adres w dokumencie - odnosimy się do id <a href = "#moj-identyfikator">

atrybut id

<element id="moj-identyfikator">

Atrybut id - identyfikuje element, przydaje się do m.in. nawigacji w obębie dokumentu

znacznik <nav>

<nav> - nawigacja po całym serwisie

znacznik <img>

<img> - obrazek o podanym adresie

atrybut src

src - source (źródło - nazwa pliku)

atrybut alt

alt - tekst alternatywny (opis obrazka - informacja dla wyszukiwarki o tym co jest na obrazku)

ATRYBUTY
  • atrybuty są niewidzialne - chowamy je w znaczniku
  • nadają elementom dodatkowe właściwości (href, src)
  • pozwalają na grupowanie elementów (class)
  • pozwalają na identyfikowanie elementów (id)
  • można je miksować

Nazewnictwo w HTML

Nazywanie rzeczy to jedna z najtrudniejszych rzeczy w programowaniu 😉

Nazywanie rzeczy:

  • nazwy zapisujemy bez polskich znaków ąćęłńóśżź
  • nie używamy spacji
  • najlepiej używać angielskich słów (treść może być w innym języku
  • trzymamy się jednego schematu (konsekwencja jest istotna)
KONWENCJE NAZEWNICTWA

Klasy CSS

atrybut style

<znacznik style="color:yellow;">treść</znacznik>

Atrybut style możemy przypiąć do danego znacznika. Jest to styl liniowy = styl inline. Jest aplikowany do znacznika, do którego został dopisany.

PRAWDZIWE SELEKTORY
#id - stylowanie za pomocą id

HTML
<znacznik id="name">treść</znacznik>
CSS
#name {
właściwość:wartość;
}

.class - stylowanie za pomocą klasy

HTML
<znacznik class="name">treść</znacznik>
CSS
.name {
właściwość:wartość;
}

element - stylowanie za pomocą elementu

HTML
<znacznik>treść</znacznik>
CSS
element {
właściwość:wartość;
}

!important

Nadpisuje wszystkie inne zdefiniowane style (nawet jeśli są osobno zdefiniowane). Lepiej nie używać, gdy się nie wie po co 😉. Oznacza się nim style CSS, dopisując !important za daną wartością.

SPECYFICZNOŚĆ W HTML

To jak tabela medalowa - złoty medal ma wyższą rangę, niż srebrny, a srebrny wyższą niż brązowy:

!important, style="", #name, .name, element

Jeśli specyficzność jest równa - pierwszeństwo ma selektor "bliżej" elementu (niżej w kodzie strony).

REKOMENDACJA KORSANA

Dodawać klasy do wszystkich elementów i unikać zagdnieżdżeń. Lepiej utworzyć klasę dla linku w nagłówku i ją ostylować. Wówczas mamy kontrolę nad tym który styl zostanie przyjęty.

NAZEWNICTWO KLAS
  • używaj pojedynczych słów z języka angielskiego,
  • nazywaj rzeczy zgodnie z ich funkcją na stronie,
  • startuj z małej litery - header nie Header,
  • nie zaczynaj od cyfr,
  • jeśli musisz użyć dwóch słów - użyj tzw. kebab-case,
  • staraj się, aby element miał jedną klasę (nie zwiększaj specyficzności bez potrzeby).
PODSUMOWANIE

- html class="name" = css .name

- html id="name" = css #name

- stylujemy po klasach, bez zagnieżdżeń!

- nazywamy klasy po angielsku, zgodnie z przeznaczeniem

- tworzymy klasy, nawet gdy mamy jeden element danego typu

- specyficzność może dać popalić 😉

- nie używamy !important i stylów inline'owych>

Jak działa internet i przeglądarka?

Protokół, który obsługuje strony internetowe i ich zawartość to HTTP (aktualnie coraz częściej HTTP/2 - mocno przyspiesza internet, ponieważ pozwala na wykonywanie równoległych połączęń i nie blokuje tak mocno serwera)

Komunikacja przeglądarka-serwer odbywa się na zasadach request-response

Odpowiedzi serwera mają różne kody i pozwalają nam na diagnozowanie błędów (zakładka Network w DevTools'ach). Kody odpowiedzi HTTP:

  • - 200 - OK
  • - 30X - szukasz w złym miejscu, przekieruję Cię w dobre miejsce (bo wiem gdzie)
  • - 404 - nie mam tego, czego potrzebujesz, sorry
  • - 50X - nie czuję się najlepiej... (błąd wewnętrzny serwera "Internal server error")

Root (/) jest ostotną częścią ścieżek do zasobów (i dobrze wiedzieć jak działa 😉)

WTF: JavaScript

- Kod JS uruchamia się raz (po załadowaniu strony)

- Java != JavaScript

- Będziemy się uczyć ECMAScript 2015+ (ES6+)

- ECMA Script to ustandaryzowany przez organizację ECMA skryptowy język programowania

- JavaScript implementuje rzeczy ze specyfikacji ECMA Script

- JavaScript to frontendowe programowanie

- Za pomocą JS można robić różne szalone rzeczy

- JavaScript składa się z:

  • wartości
  • literały (ang. literal)
  • operacje porównywania
  • zmienne
  • funkcje
  • obiekty
